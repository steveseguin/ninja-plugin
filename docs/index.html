OBS + VDO.Ninja Plugin

Publish and ingest VDO.Ninja streams directly in OBS

Native WebRTC path for OBS output publishing and inbound stream handling, with multi-viewer support, data channel metadata, and release-ready binaries.

Download Latest Release View Repository

What It Does

  • Publishes OBS output to VDO.Ninja using push stream IDs.
  • Ingests VDO.Ninja streams into OBS via source mode or auto-inbound management.
  • Handles multiple viewers from a single publisher session.
  • Parses data-channel metadata (including WHEP URL hints).

Advantages

  • Lower operational overhead than managing many Browser Sources manually.
  • Built-in OBS workflow for stream IDs, passwords, room IDs, and reconnects.
  • Useful for remote contribution, director-style workflows, and room ingestion.
  • Open-source with CI-tested Linux, Windows, and macOS builds.

How To Use (Quick Start)

  1. Download and extract the release package, then run install.cmd (Windows) or install.sh (Linux/macOS).
  2. Windows tip: use install.cmd first. It handles strict PowerShell policy setups.
  3. In OBS, open Settings -> Stream and pick VDO.Ninja.
  4. Set Stream ID and optional Password / Room ID, then start streaming.
  5. Open your viewer URL: https://vdo.ninja/?view=YourStreamID (add password if set).
  6. For ingest, add a VDO.Ninja Source in OBS and enter the same IDs.
Example:
Publish: https://vdo.ninja/?push=MyStreamID&password=MyPass
View:    https://vdo.ninja/?view=MyStreamID&password=MyPass

Advanced Settings

Supports custom salt/domain behavior and custom signaling / ICE configuration for hosted or constrained environments.

  • Custom salt (default VDO.Ninja compatible behavior).
  • Custom websocket signaling server URL.
  • Custom STUN/TURN servers and relay preferences.
  • Viewer limits and auto-inbound source controls.

Status

  • Native source decode path exists and is still being hardened.
  • Locale fallback to English is supported if locale files are missing.
